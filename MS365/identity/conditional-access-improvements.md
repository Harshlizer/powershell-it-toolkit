# Conditional Access Improvements

## Summary

This note captures several practical improvements that would make Conditional Access easier to operate, safer to govern, and easier to understand.

## Key Gaps

- Deny-by-default behavior instead of allow-by-default
- Built-in change audit with actor, timestamp, and exact delta
- Native policy documentation fields
- Human-readable if-then summaries
- Better import, export, and comparison workflows
- Easier application targeting
- Stronger separation for Protected Actions
- Explicit `AND` / `OR` condition logic
